# RAG 流水线

## 概述

RAG 流水线被定义为由 `@rag-sdk/runtime` 执行的步骤组成的有向无环图（DAG）。流水线编排从文档加载到答案生成的完整 RAG 生命周期。

## 流水线生命周期

```
                  ┌──────────────────┐
                  │    文档加载       │
                  └────────┬─────────┘
                           │
                  ┌────────▼─────────┐
                  │    文档分块       │
                  └────────┬─────────┘
                           │
                  ┌────────▼─────────┐
                  │    Embedding     │
                  └────────┬─────────┘
                           │
                  ┌────────▼─────────┐     ┌─────────────────┐
                  │  向量存储写入     │────▶│   索引完成       │
                  └────────┬─────────┘     └─────────────────┘
                           │
                    （查询时）
                           │
                  ┌────────▼─────────┐
                  │   查询 Embedding │
                  └────────┬─────────┘
                           │
                  ┌────────▼─────────┐
                  │    向量搜索       │
                  └────────┬─────────┘
                           │
                  ┌────────▼─────────┐
                  │  上下文窗口构建   │
                  └────────┬─────────┘
                           │
                  ┌────────▼─────────┐
                  │   LLM 生成       │
                  └────────┬─────────┘
                           │
                  ┌────────▼─────────┐
                  │     响应输出     │
                  └──────────────────┘
```

## 流水线步骤

### 1. 文档加载
- **包：** `@rag-sdk/indexing`
- **输入：** 原始文档（PDF、HTML、Markdown、纯文本、代码等）
- **输出：** 规范化的 `Document` 对象及元数据
- **关注点：** 文档加载器、格式检测、元数据提取

### 2. 文档分块
- **包：** `@rag-sdk/indexing`
- **输入：** `Document` 对象
- **输出：** `Chunk` 对象数组
- **策略：**
  - 固定大小分块（字符数 + 重叠）
  - 递归分割（按段落、句子、分隔符）
  - 语义分块（按 embedding 相似度边界）

### 3. Embedding（索引阶段）
- **包：** `@rag-sdk/adapters` + `@rag-sdk/indexing`
- **输入：** `Chunk[]`
- **输出：** 附加了 embedding 的 `Chunk[]`
- **关注点：** 批处理、速率限制、embedding 模型选择

### 4. 向量存储写入
- **包：** `@rag-sdk/adapters`
- **输入：** 已嵌入的分块
- **输出：** 向量已索引（副作用）
- **关注点：** 批量 upsert、命名空间管理、元数据过滤

### 5. 查询 Embedding
- **包：** `@rag-sdk/adapters`
- **输入：** 用户查询字符串
- **输出：** 查询 embedding 向量

### 6. 向量搜索
- **包：** `@rag-sdk/adapters`
- **输入：** 查询 embedding + 搜索参数
- **输出：** 带相似度分数的排序 `Chunk[]`
- **参数：** topK、相似度阈值、过滤条件

### 7. 上下文窗口构建
- **包：** `@rag-sdk/runtime`
- **输入：** 排序后的分块 + 查询
- **输出：** 格式化的上下文字符串
- **关注点：** Token 预算管理、分块排序、相关性过滤

### 8. LLM 生成
- **包：** `@rag-sdk/adapters` + `@rag-sdk/runtime`
- **输入：** 系统提示词 + 上下文 + 用户查询
- **输出：** 生成的答案
- **关注点：** 流式输出、重试、Token 限制、结构化输出

## 流水线配置

流水线通过声明式方式配置：

```typescript
import { Pipeline } from '@rag-sdk/runtime';

const pipeline = new Pipeline({
  name: 'qa-pipeline',
  steps: [
    { name: 'chunk', config: { strategy: 'recursive', chunkSize: 1000 } },
    { name: 'embed', config: { provider: 'openai', model: 'text-embedding-3-small' } },
    { name: 'retrieve', config: { topK: 5, minScore: 0.7 } },
    { name: 'generate', config: { model: 'gpt-4o', temperature: 0.1 } },
  ],
});
```

## 错误处理

- **每步重试：** 可配置的重试策略，指数退避
- **优雅降级：** 流水线可以使用部分结果继续执行
- **可观测的失败：** 所有错误在链路追踪中被捕获
