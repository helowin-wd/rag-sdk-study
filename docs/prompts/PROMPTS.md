# 提示词管理

## 概述

提示词管理作为 SDK 运行时层的一部分处理。提示词被视为**模板**，在流水线执行时组合，而非硬编码的字符串。

本文档描述了 RAG SDK 的提示词架构。具体实现将随着 `@rag-sdk/runtime` 包的开发而跟进。

## 设计原则

1. **模板优先于字符串** — 提示词是带有变量注入的结构化模板
2. **可组合** — 系统提示词、上下文模板和输出格式是可组合的
3. **可观测** — 所有提示词构建都会被记录，用于调试和评估
4. **可版本化** — 提示词模板与代码一同追踪

## 提示词类型

### 系统提示词

定义 LLM 的行为和约束：

```
你是一个乐于助人的助手，专门根据提供的上下文回答问题。
请遵循以下规则：
1. 仅使用提供的上下文回答
2. 如果上下文中不包含答案，请明确说明
3. 使用 [来源: 文档名称] 引用来源
4. 保持答案简洁明了
```

### 上下文模板

格式化检索到的分块供 LLM 使用：

```
上下文：
{context}

---
文档：
{documents}
```

### 查询模板

结构化用户查询：

```
基于以上上下文，请回答：
{query}

要求：
- {additional_instructions}
```

### 输出格式提示词

引导 LLM 输出结构：

```
请以 JSON 对象格式提供答案，包含以下字段：
- "answer": 字符串（你的回答）
- "sources": 字符串数组（来源文档名称列表）
- "confidence": "high" | "medium" | "low"
```

## 模板变量

变量在运行时由流水线注入：

| 变量 | 来源 | 描述 |
|----------|------|-------------|
| `{query}` | 用户输入 | 用户的问题 |
| `{context}` | 流水线步骤 | 来自检索分块的格式化上下文 |
| `{documents}` | 流水线步骤 | 来源文档引用 |
| `{history}` | 流水线状态 | 对话历史（用于多轮对话） |
| `{additional_instructions}` | 流水线配置 | 动态指令 |

## 提示词组合流程

```
用户查询
    │
    ▼
┌─────────────────────┐
│  查询模板            │  例如："回答：{query}"
│  + 上下文模板        │  例如："上下文：{context}"
│  + 系统提示词        │  例如："你是一个专家..."
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│  最终提示词          │  发送给 LLM 的组合提示词
│  （可观测）          │  记录用于调试/评估
└─────────────────────┘
```

## 实现计划

1. 在 `@rag-sdk/core` 中定义 `PromptTemplate` 接口
2. 在 `@rag-sdk/runtime` 中实现模板渲染
3. 添加上下文窗口 Token 管理
4. 构建提示词调试工具（发送前查看组合后的提示词）
5. 在 `@rag-sdk/eval` 中添加提示词版本管理和 A/B 测试支持
