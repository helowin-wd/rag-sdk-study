# 架构说明

## 概述

RAG SDK 采用**分层模块化架构**，基于 pnpm workspaces 的 monorepo 构建。每个包拥有单一职责，通过 `@rag-sdk/core` 中定义的接口与其他包通信。

## 分层结构

```
┌─────────────────────────────────────────────────┐
│                   应用层                          │
├─────────────────────────────────────────────────┤
│                      SDK                         │
├─────────────────────────────────────────────────┤
│   @rag-sdk/runtime                               │
│   （流水线编排与执行）                              │
├──────────────┬──────────────────┬───────────────┤
│ @rag-sdk/    │ @rag-sdk/        │ @rag-sdk/      │
│ indexing     │ adapters         │ eval           │
│ （文档加载、  │ （LLM、Embedding、│ （评估指标、    │
│  分块、索引） │  向量存储）       │  测试框架）     │
├──────────────┴──────────────────┴───────────────┤
│              @rag-sdk/core                       │
│   （类型、接口、基础抽象）                          │
├─────────────────────────────────────────────────┤
│   @rag-sdk/utils            @rag-sdk/            │
│   （共享工具函数）            observability        │
│                              （链路追踪、日志）     │
├─────────────────────────────────────────────────┤
│              运行时层（Node.js）                    │
└─────────────────────────────────────────────────┘
```

## 包依赖关系

```
@rag-sdk/utils ─────────────────────────────────┐
                                               │
@rag-sdk/core ───────┬──────────┬──────────────┤
                     │          │              │
              @rag-sdk/   @rag-sdk/      @rag-sdk/
              adapters    indexing        eval
                     │          │              │
                     └──────────┴──────────────┘
                               │
                        @rag-sdk/runtime
                               │
                        @rag-sdk/observability
```

- `@rag-sdk/core` — 基础层，不依赖其他工作区包
- `@rag-sdk/utils` — 基础设施层，不依赖工作区包
- `@rag-sdk/adapters`、`@rag-sdk/indexing`、`@rag-sdk/eval` — 依赖 `@rag-sdk/core`
- `@rag-sdk/runtime` — 编排层，依赖 core + indexing + adapters
- `@rag-sdk/observability` — 横切关注点，依赖 core

## 设计原则

### 1. 接口优先

所有跨包通信通过 `@rag-sdk/core` 中定义的接口进行。这使得每个包可以独立开发和测试。

### 2. 提供商抽象（适配器模式）

`@rag-sdk/adapters` 包使用适配器模式抽象以下内容：
- **LLM 提供商**（OpenAI、Anthropic 等）
- **Embedding 提供商**（OpenAI、Cohere 等）
- **向量存储**（Pinecone、Weaviate、pgvector 等）

每个适配器实现 `@rag-sdk/core` 中定义的通用接口，使得更换提供商无需修改流水线逻辑。

### 3. 可组合流水线

`@rag-sdk/runtime` 包将流水线定义为有向无环图（DAG）的步骤。每个步骤是一个自包含单元，可以独立组合、测试和观测。

### 4. 默认可观测

每个流水线步骤通过 `@rag-sdk/observability` 发出链路追踪和指标。这是在运行时层面配置的，而非事后添加。

## 关键架构决策

| 决策 | 理由 |
|----------|-----------|
| 使用 pnpm workspaces 的 Monorepo | 共享类型、单一版本管理、简化跨包开发 |
| 仅支持 ESM 模块 | 更现代、更好的 tree-shaking、与 TypeScript 生态一致 |
| Core 作为独立包 | 避免循环依赖、强制接口边界 |
| 提供商使用适配器模式 | 新增提供商无需修改流水线逻辑 |
| 基于 DAG 的流水线执行 | 支持复杂分支、并行执行和可观测性 |
