# 项目背景

## 项目简介

本项目是一个 **学习型项目**，目标是从头构建一个生产级的 RAG（检索增强生成）SDK。通过自己设计和实现每个组件，深入理解 RAG 系统的完整生命周期。

## 动机

RAG 是现代 LLM 应用中的核心模式。虽然有现成的 SDK（LangChain、LlamaIndex），但从头构建一个能够带来对这些领域的深入理解：

1. **文档加载流水线** — 解析、分块、元数据提取
2. **嵌入和向量存储** — 与各种提供商的集成模式
3. **LLM 交互模式** — 提示词构建、上下文注入、响应生成
4. **评估** — 衡量检索质量、响应质量、端到端性能
5. **可观测性** — 在生产环境中追踪 RAG 流水线
6. **模块化 SDK 设计** — 创建清晰的抽象和可组合的 API

## 目标

### 主要目标

- 设计一个具有清晰接口的模块化 RAG SDK
- 实现完整的 RAG 流水线：加载 → 索引 → 检索 → 生成
- 通过统一的适配器接口支持多种 LLM、Embedding 和向量存储提供商
- 构建用于衡量 RAG 质量的评估框架
- 将可观测性（链路追踪、指标）作为一等公民

### 非目标

- 不旨在替代 LangChain 或 LlamaIndex 等成熟 SDK
- 不关注生产部署（现阶段无 Docker、无 CI/CD）
- 不构建向量数据库 — 我们集成现有的

## 范围

SDK 涵盖以下领域：

| 领域 | 包 | 状态 |
|--------|---------|--------|
| 核心类型与接口 | `@rag-sdk/core` | 已搭建脚手架 |
| 提供商适配器 | `@rag-sdk/adapters` | 已搭建脚手架 |
| 文档索引 | `@rag-sdk/indexing` | 已搭建脚手架 |
| 流水线运行时 | `@rag-sdk/runtime` | 已搭建脚手架 |
| 评估 | `@rag-sdk/eval` | 已搭建脚手架 |
| 可观测性 | `@rag-sdk/observability` | 已搭建脚手架 |
| 共享工具 | `@rag-sdk/utils` | 已搭建脚手架 |

## 目标用户

- 希望使用轻量级、类型安全 SDK 构建 RAG 应用的**开发者**
- 通过参考实现学习 RAG 架构的**学生/学习者**
- 对模块化 SDK 设计模式感兴趣的**贡献者**

## 关键约束

- **仅 TypeScript / ESM** — 不计划支持 CommonJS
- **最小外部依赖** — 优先自建而非引入
- **工作区 Monorepo** — 所有包位于单一仓库中
