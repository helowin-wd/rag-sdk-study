# 项目初始化需求文档

## 1. 项目目标

本阶段仅完成：

> **创建 RAG SDK 的 Monorepo 目录结构**

不涉及任何业务逻辑、功能实现或复杂配置。

------

## 2. 技术要求

- 使用 `pnpm + workspace`
- 使用 TypeScript（仅基础配置）
- 所有 package 必须为独立包（包含 `package.json`）

------

## 4. Package 要求

`packages/` 下的每一个目录：

必须包含：

```text
package.json
```

👉 不要求：

- 不需要实现代码
- 不需要 src 目录
- 不需要任何逻辑

------

## 5. workspace 配置

需要提供：

### pnpm-workspace.yaml

```yaml
packages:
  - "packages/*"
```

------

## 6. 严格限制（必须遵守）

本阶段：

### ✅ 允许

- 创建目录
- 创建 package.json
- 初始化 workspace

------

### ❌ 禁止

- 写任何业务代码
- 写 RAG 逻辑
- 写接口实现
- 引入第三方依赖
- 设计复杂配置
- 添加无关目录

------

## 7. 验收标准

满足以下即可：

- `pnpm install` 能正常执行
- `packages/*` 下所有目录都有 `package.json`
- 目录结构与要求一致

------

## 8. 一句话总结

> 本阶段只做一件事：**把 RAG SDK 的工程骨架搭出来，不写任何功能**
