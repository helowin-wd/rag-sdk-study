# 安装包

## 1. 哪些包应该安装在“根目录”？

通常是那些**开发工具类**、**全局构建类**的包。

- **TypeScript (`typescript`)**：必须在根目录，保证整个项目使用同一个编译版本，避免包与包之间因为 TS 版本不一致产生的类型冲突。
- **构建/代码质量工具**：如 `eslint`, `prettier`, `vitest`, `tsup`（如果你用它打包）。
- **Workspace 管理工具**：如 `pnpm` 相关的插件。

**安装命令：**

```bash
pnpm add typescript -w -D
```

> `-w` 代表 `workspace-root`，`-D` 代表 `devDependencies`。

------

## 2. 哪些包应该安装在“子包（packages/xx）”？

即使所有子包都用到了某个功能，如果它是**运行时（Runtime）**依赖，也建议安装在子包中。

- **Zod**：虽然 `core`、`runtime` 都会用到，但建议分别在各子包内安装。
- **工具函数库**（如 `lodash-es`）：安装在子包中。

**原因：** 1. **显式依赖**：每个包的 `package.json` 应该能独立描述运行它需要什么，这样发布到 NPM 后，用户安装你的 `core` 包时，才能自动下载 `zod`。 2. **pnpm 特性**：pnpm 会使用符号链接（Symbolic Links）处理重复依赖。即使你在 5 个子包里都装了 `zod`，在磁盘上也只有一份物理存储，不会浪费空间。



## 3. 总结

在 Monorepo 中，依赖管理遵循“谁使用，谁声明”的原则。

开发工具（如 TypeScript、Vitest、ESLint）统一安装在根目录。

运行时依赖（如 zod、openai、lodash）必须安装在实际使用它的子包中，不能依赖其它包的隐式依赖。

每个子包的 package.json 必须能独立描述其运行依赖，以保证发布后的可用性。

pnpm 会自动去重依赖，因此无需担心重复安装带来的空间问题。
